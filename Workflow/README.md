# 新闻编辑工作流

固定采集与本地 Codex/兼容模型增强彼此独立。没有任何 AI 凭据时，GitHub Actions 和客户端内置采集器仍能完成采集、去重、来源限额、时间窗口与发布；AI 只负责在来源材料范围内补充中文解释。

工作流按固定阶段运行，避免把“搜索到什么就直接展示”混成一个不可检查的脚本。

1. **读取偏好**：读取 `%LOCALAPPDATA%\AIFrontier\workflow-profile.json`；不存在时使用 `Data/preference-profile.json`。
2. **候选采集**：覆盖大模型、Agent、开源项目、重要研究、产业动态和人物观点，主窗口为近 72 小时。
3. **来源分层**：官方公告、论文、GitHub 优先；媒体和社交平台只作为发现线索。
4. **事实核查**：校验事件发生时间、原始链接和关键数字，删除重复与未经证实内容。
5. **偏好排序**：主题权重决定排序，不改变事实判断和来源门槛。
6. **入门改写**：每条分别生成摘要、关键事实、背景、入门解释、影响、局限、后续观察与原始来源，避免同一句话在多个字段重复。
7. **结构校验**：验证 `Data/news.json` 的必需字段、链接与条目数量。
8. **发布同步**：原生客户端读取 JSON；公共网站可从同一份数据生成，二者不再维护两套文案。

## 独立运行

```powershell
python scripts/update_feed.py --config Data/source-feeds.json --output Data/news.json
```

该命令只使用 Python 标准库。设置 `AI_API_KEY`、`AI_API_BASE` 和 `AI_MODEL` 后才会启用兼容接口的编辑增强；任一变量缺失时自动回退，不影响采集。

## 反馈映射

- 喜欢：对应主题 `+0.12`，来源 `+0.06`。
- 不感兴趣：对应主题 `-0.12`，来源 `-0.04`。
- 减少此类：对应主题 `-0.25`。
- 收藏：不强行改变主题，只把该条加入收藏并记录阅读深度信号。
- 权重限制在 `0.2–2.0`，防止短期连续点击让信息流失真。
